HashMap

简介

  • 用于存放键值对

  • 基于哈希表的Map接口实现

  • 不支持数据保存的顺序

  • 非线程安全

JDK1.8之前底层为链表散列,1.8之后将当 链表长度大于阈值（默认为 8）时，将链表转化为红黑树 .

( 将链表转换成红黑树前会判断，如果当前数组的长度小于 64，那么会选择先进行数组扩容，而不是转换为红黑树），以减少搜索时间 )

Q:什么是链表散列?

链表散列为一种数组+链表的结构.

Q:为什么用链表散列?

为了解决哈希冲突

Q:什么是哈希冲突?

哈希算法的目的就是努力的把比较大的数据存放到相对较小的空间中.

为了实现这个目的就需要将数据通过计算将其放在一个有限的区间内,比如将1–10000通过计算放在1–100这个区间.

这样就会重复(计算后两个数值一样)这就形成了哈希冲突

Q:怎么解决?

将重复的数据,经过哈希处理后,以链表的形式放入对应的链表中.(拉链法)

底层数据结构

散列链表

HashMap 通过 key 的 hashCode 经过扰动函数处理过后得到 hash 值，然后通过 (n - 1) & hash 判断当前元素存放的位置（这里的 n 指的是数组的长度），

如果当前位置存在元素的话，就判断该元素与要存入的元素的 hash 值以及 key 是否相同，如果相同的话，直接覆盖，不相同就通过拉链法解决冲突。

所谓扰动函数指的就是 HashMap 的 hash 方法。使用 hash 方法也就是扰动函数是为了防止一些实现比较差的 hashCode() 方法 换句话说使用扰动函数之后可以减少碰撞。

类属性

Q:为什么加载因子为0.75?

A: 加载因子设置为0.75而不是1，是因为设置过大，桶中键值对碰撞的几率就会越大，同一个桶位置可能会存放好几个value值，这样就会增加搜索的时间，性能下降，设置过小也不合适

Q:为什么hashMap的大小是2的n次?

A: HashMap为了存取高效，要尽量较少碰撞，就是要尽量把数据分配均匀，每个链表长度大致相同，这个实现就在把数据存到哪个链表中的算法；
这个算法实际就是取模，hash%length，计算机中直接求余效率不如位移运算，源码中做了优化hash&(length-1)，
hash%length==hash&(length-1)的前提是length是2的n次方；

Q:为什么这样能均匀分布减少碰撞呢？

A:2的n次方实际就是1后面n个0，2的n次方-1 实际就是n个1；

Q:threshold的计算公式

A: threshold = loadFactor * capacity
