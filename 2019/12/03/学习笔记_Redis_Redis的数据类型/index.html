学习笔记_Redis_Redis的数据类型

与MySQL不同,Redis中的数据是以键值对的形式存在的

Redis

Redis的数据类型

在redis中的命令语句中，命令是忽略大小写的，而key是不忽略大小写的。

首先 选择数据库

  • 命令:select 数据库编号

关于Key

dbsize 查看当前数据库的key的数量

flushdb 清空当前数据库

flushall 通杀全部库

expire <key> <seconds> 为键值设置过期时间,单位为秒

ttl <key> 查看还有多少秒过期,-1表示永不过期,-2表示已经过期

注意:redis的命令具有原子性

String类型

  • 赋值

    语法:SET key value

  • 取值

    语法:GET key

  • 设置/获取多个值

    语法:

  • 取值并赋值

    语法:GETSET key value

    允许key不存在,key不存在的话,取值为nil

  • 删除

    语法:DEL key

    结果为受影响行数

  • 数值增减

    • 数值递增(实现指定key的值增长)

      语法:INCR key

  • 增长指定整数

    语法:INCERBY key increment

  • 数值递减

    • 数值递减

      语法:DECR key

  • 减少指定整数

    语法:DECERBY key increment

  • 向尾部追加值

    APPEND的作用是向键值的末尾追加value。如果键不存在则将该键的值设置为value，即相当于 SET key value。返回值是追加后字符串的总长度。

    语法：APPEND key value

  • 获取字符串长度

    STRLEN命令返回键值的长度，如果键不存在则返回0。

    语法：STRLEN key

补充:

setnx <key> <value> 只有在key不存在的时候设置key的值

incrby/decrby <key> <步长> 将key中存储的数字值增减,自定义步长

msetnx <key1> <value1> <key2> <value2> ... 同时设置一个或多个key-value对,当且仅当所有给定key都不存在.

getrange <key> <起始位置> <结束位置> 获得值的范围,类似java中的substring

setrange <key <起止位置> <value> 用<value>复写<key>所存储的字符串值,从<起始位置>开始.

setex <key> <过期时间> <value> 设置键值的同时,设置过期时间,单位秒.

Hash类型

hash叫散列类型，它提供了字段和字段值的映射。字段值只能是字符串类型，不支持散列类型、集合类型等其它类型。

  • 赋值

    • 一次设置一个

      语法:HSET key field value

      HSET命令不区分插入和更新操作，当执行插入操作时HSET命令返回1，当执行更新操作时返回0。

    • 一次设置多个

      语法:HMSET key field value [field value ...]

  • 取值

    • 一次获取一个值

      语法:HGET key field

    • 一次获取多个值

      语法:HMGET key field [field ...]

    • 获取所有字段值

      语法:HGETALL key

  • 删除字段

    可以删除一个或多个字段，返回值是被删除的字段个数

    语法:HDEL key field [field ...]

  • 增加数字

    语法:HINCRBY key field increment

  • 判断字段是否存在

    语法:HEXISTS key field

  • 只获取字段名或字段值

    语法:HKEYS key

    ​ HVALS key

  • 获取字段数量

    语法:HLEN key

list类型

​ 列表类型（list）可以存储一个有序的字符串列表，常用的操作是向列表两端添加元素，或者获得列表的某一个片段。

​ 列表类型内部是使用的是双向链表（double linked list），所以向列表两端添加元素的时间复杂度为0(1)，获取越接近两端的元素速度就越快。这意味着即使是一个有几千万个元素的列表，获取头部或尾部的10条记录也是极快的。

  • 向列表两端增加元素

    • 向列表左边增加元素

      LPUSH key value [value ...]

    • 向列表右边增加元素

      RPUSH key value [value ...]

  • 查看列表

    语法:LRANGE key start stop

    将返回start、stop之间的所有元素（包含两端的元素），索引从0开始。索引可以是负数，如：“-1”代表最后边的一个元素。

  • 从列表两端弹出元素

    • 从左边弹出

      LPOP key

    • 从右边弹出

      RPOP key

    返回值为弹出的值

    值在键在,值亡键亡

  • 获取列表中元素的个数

    语法:LLEN key

  • 删除列表指定的值

    语法:LREM key count value

    LREM命令会删除列表中前count个值为value的元素，返回实际删除的元素个数。根据count值的不同，该命令的执行方式会有所不同：

    • 当count>0时， LREM会从列表左边开始删除一个。

    • 当count<0时， LREM会从列表后边开始删除一个。

    • 当count=0时， LREM删除所有值为value的元素。

  • 获得指定索引的元素值

    语法:LINDEX key index

  • 设置指定索引的元素值

    语法:LSET key index value

  • 只保留列表指定片段

    语法:LTRIM key start stop

  • 向列表中插入元素

    该命令首先会在列表中从左到右查找值为pivot的元素，然后根据第二个参数是BEFORE还是AFTER来决定将value插入到该元素的前面还是后面.

    语法:LINSERT key BEFORE|AFTER pivot value

  • 将元素从一个列表转移到另一个列表中

    语法：RPOPLPUSH source destination

  • 按照索引下标获得元素(从左向右)

    语法:LRANGE <key> <start> <stop>

  • 按照索引下标获得元素(从左向右)

    语法:LINDEX <key> <value>

Set类型

  • 增加元素

    语法:SADD key member [member1 ...]

  • 删除元素

    语法:SREM key member [member2 ...]

  • 获取集合中的所有元素

    语法:SMEMBERS key

  • 判断元素是否在集合中

    语法:SISMEMBER key member

  • 运算相关

    • 集合的差集运算A-B

      (属于A并且不属于B的元素构成的集合)

      语法:SDIFF keyA keyB

  • 集合的交集运算A∩B

    (属于A且属于B的元素)

    语法:SINTER keyA keyB

  • 集合的并集运算A∪B

    (属于A或者属于B)

    语法:SUNION keyA keyB

  • 获得集合中元素大的个数

    语法:SCRAD key

  • 从集合中弹出一个元素

    语法:SPOP key

SortedSet类型zset

有序的集合,相较于set而言更消耗性能

  • 添加元素

    语法:ZADD key score member [score member]

  • 获取元素的分数(在Redis中的顺序)

    语法:ZSOCRE key member

  • 删除元素

    语法:ZREM key member [member ...]

    移除有序集key中的一个或多个成员，不存在的成员将被忽略。

    当key存在但不是有序集类型时，返回一个错误。

  • 获得排名在某个范围的元素列表

    语法：ZRANGE key start stop [WITHSCORES]

    按照元素分数从小到大的顺序返回索引从start到stop之间的所有元素（包含两端的元素）如果需要获得元素的分数的可以在命令尾部加上WITHSCORES\参数

  • 其它命令

    • 获得指定分数范围的元素

      语法：ZRANGEBYSCORE key min max [WITHSCORES] [LIMIT offset count]

  • 增加某个元素的分数

    语法：ZINCRBY key increment member

    返回值是更改后的分数

  • 获得集合中元素的数量

    语法：ZCARD key

  • 获得指定分数范围内的元素个数

    语法：ZCOUNT key min max

  • 按照排名范围删除元素

    语法：ZREMRANGEBYRANK key start stop

  • 按照分数范围删除元素

    语法：ZREMRANGEBYSCORE key min max

  • 获取元素的排名

    从小到大 语法：ZRANK key member

    从大到小 语法：ZREVRANK key member
